05 CASE STUDIES
CASE 01 / FINANCIO

From hidden hover to actionable status.

Problem

The e-invoice status lived behind a hover pop-up — low visibility, no clarity, and no path to resolve an issue when something went wrong.

Approach

Mapped user behaviour around the status check to find where confusion and support dependency were coming from, then focused the redesign on visibility and actionability.

Solution

Replaced the hover with a modal built around a stepper — clear stage indicators including "In Progress," integrated error messaging, and the redundant QR code removed.

Impact

  • Status became visible without guesswork
  • Fewer support escalations for "is this invoice valid?"
  • Errors are now actionable, not just visible
CASE 02 / FINANCIO

RinggitPay × Financio integration.

Problem

Initial scope covered only the invoice screen, on the assumption that payment happens at invoice level — a framing that risked breaking the experience everywhere else it touched.

Approach

Ran an impact analysis with the RinggitPay team and traced the integration across the product, surfacing seven affected areas instead of one.

Solution

Designed end-to-end flows and prototypes for every touchpoint — invoice, notifications, contacts, integration settings, account module, payment receipts, and Financio Connect — validated with developers and QA before sign-off.

Impact

  • One coherent payment experience, not seven patched-together screens
  • No rework after dev handoff — gaps were caught pre-build
  • Set the pattern the team now uses for scoping integrations
CASE 03 / FINANCIO — SINGAPORE

Simplifying advanced ordering for InvoiceNow (IMDA).

Problem

Managing order changes — cancellations, edits, balance updates — was fragmented across Singapore's InvoiceNow network, raising both confusion and error risk.

Approach

Researched real PO workflows to find the pain points, then worked with product, BA and dev to keep the fix aligned with backend constraints.

Solution

A single "golden feature" that consolidates cancellations, adjustments, and edits into one interface — delivered through wireframes and iterative validation.

Impact

  • Order changes live in one place instead of scattered flows
  • Lower error rate on balance and cancellation edits
  • UX shaped by, not fighting, technical feasibility
CASE 04 / FINANCIO MOBILE

Bringing LHDN e-invoicing to mobile.

Problem

E-invoice submission was desktop-only, which cut off anyone needing to invoice while out in the field.

Approach

Worked with developers to test technical feasibility and pinned down which LHDN fields were actually mandatory, so the existing system stayed untouched.

Solution

A standalone mobile submission flow carrying only the essential fields — fast enough for on-the-go invoicing without a parallel system to maintain.

Impact

  • Invoicing no longer tied to a desk
  • Faster submission with a reduced field set
  • Shipped without disrupting the existing web flow
CASE 05 / ABSS

Elevating the download experience for ABSS.

Problem

After purchase, users landed on a bare SharePoint link — a jarring disconnect from a paid, premium product.

Approach

Researched where the trust gap came from and benchmarked app-store-style download experiences as a reference point.

Solution

An app-store-inspired download page with product detail, feature highlights, and a guided flow from purchase to install.

Impact

  • Download now feels like part of the product, not an afterthought
  • Less hesitation and fewer "did my purchase go through?" queries
  • Consistent premium feel across the purchase-to-install journey
